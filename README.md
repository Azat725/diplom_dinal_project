Предварительные требования
Установленный Python (версия 3.8 или выше)

Установленный Git (опционально, для клонирования репозитория)

Виртуальное окружение (рекомендуется)

Шаги по установке
1. Получите код проекта
Вариант А: Клонировать репозиторий (если у вас есть Git)

bash
git clone https://github.com/Azat725/diplom_dinal_project.git
cd diplom_dinal_project
Вариант Б: Скачать архив

Перейдите по ссылке: https://github.com/Azat725/diplom_dinal_project

Нажмите кнопку Code → Download ZIP

Распакуйте архив и перейдите в папку проекта через терминал:

bash
cd путь/к/распакованной/папке
2. Создайте и активируйте виртуальное окружение
Windows

bash
python -m venv venv
venv\Scripts\activate
macOS / Linux

bash
python3 -m venv venv
source venv/bin/activate
3. Установите необходимые пакеты
В проекте используются следующие зависимости:

Django

django-ckeditor

Pillow

Установите их одной командой:

bash
pip install django django-ckeditor Pillow
Примечание: Если в будущем появятся ошибки об отсутствии других модулей, установите их аналогично: pip install <имя_модуля>.

После установки всех пакетов рекомендуется создать файл requirements.txt для фиксации зависимостей:

bash
pip freeze > requirements.txt
В следующий раз установить их можно будет командой pip install -r requirements.txt.

4. Выполните миграции базы данных
bash
python manage.py migrate
5. (Опционально) Создайте суперпользователя для доступа в админ-панель
bash
python manage.py createsuperuser
Следуйте инструкциям в терминале (введите имя, email и пароль).

6. Запустите сервер разработки
bash
python manage.py runserver
Вы увидите сообщение, подобное:

text
Starting development server at http://127.0.0.1:8000/
Откройте браузер и перейдите по адресу http://127.0.0.1:8000/, чтобы увидеть работающее приложение.

Возможные проблемы и их решение
Ошибка ModuleNotFoundError: No module named '...'
Установите недостающий модуль:

bash
pip install <имя_модуля>
Предупреждение ckeditor.W001 о безопасности CKEditor
Это предупреждение не влияет на работу приложения в режиме разработки. Для production-проекта рекомендуется обновить редактор или перейти на django-ckeditor-5 (с проверкой лицензионных условий).

Дополнительная информация
Админ-панель доступна по адресу http://127.0.0.1:8000/admin/ (если вы создали суперпользователя).

Файл базы данных по умолчанию — db.sqlite3 (уже включён в репозиторий).
